### [LOW] `tests/db/queries.test.ts` 타임존 경계 — 자정 경합
- **File**: `tests/db/queries.test.ts:194-201`
- `day1 = now - 2d`, `day2 = now - 1d`를 로컬 clock으로 잘라 `slice(0,10)`. SQLite의 `date('now', '-30 days')`는 UTC 기준. KST 런너에서 00:00~09:00 UTC 윈도우에 돌 때 `day1/day2`가 UTC day와 엇갈릴 수 있음(실제 SQL 비교는 문자열이라 현재 30-day 윈도우 내에 항상 들어와서 green 유지될 것이나 의미가 흐려짐).
- **권장**: `new Date().toISOString().slice(0,10)`을 기준값으로 쓰고 거기서 epoch day 계산(이미 다른 곳에서 사용 중인 패턴과 일치). 현재 그대로도 green이라 LOW.
